<big><strong>GPIO(General Use Input/Output) ，通用输入输出，可以被配置为输入或输出，以满足不同的应用需求。</strong></big><br>

# 开发环境配置

1. 选择**STM32CubeIDE**（本质STM32CubeMX+eclipse）<br>

2. 选择**STM32CubeMX**生成基础代码，并配合其他编辑器使用：<br>
- CLion：强大的C语言编辑功能，以及官方对嵌入式开发的适配，使其成为开发的利器；<br>
- Keil：调试的灵活性，使它能够胜任许多复杂的工作<small>:spoiler[但是真的丑爆了！qwq]</small>
- VSCode：众多的社区插件，让VSCode也能作为顺手的工具。<br>
    - 若使用**CMake**：[参考部署视频](https://www.bilibili.com/video/BV1QfbpzGENy/)，<br>
    - 若搭配Keil使用**MDK-ARM**：[参考部署视频](https://www.bilibili.com/video/BV19V411g7gD)；<br><br>

# 芯片配置
打开STM32CubeMX，选择需要的芯片并创建工程（以STM32F103C8T6为例）：<br>
1. 左键芯片引脚，可以选择GPIO口负责的功能，例如**GPIO_Input**可以从外部捕获高低电平，**GPIO_Output**可以对外输出高低电平。<br>

2. 设置对应功能后，可右键引脚，选择**Enter User Label**，可以为引脚自定义名称，便于之后调用。例如：命名为`LED_Red`，之后调用时可输入`LED_Red_GPIO_Port`和`LED_Red_Pin`<br>
<img src="/images/posts/STM32/STM32-1/cubemx0.png" width="400" height="400">

3. 在左侧选择**System Core**展开后选择**SYS**，在其中的**GPIO**选项可以调整GPIO口的具体信息：<br>
    - `GPIO output level`：默认输出高电平/低电平；<br>
    - `GPIO mode`：输出模式，推挽输出/开漏输出；<br>
    - `GPIO Pull-up/Pull-down`：选择上拉模式/下拉模式/浮空输入。<br>

4. 在左侧选择**System Core**展开后选择**SYS**，在其中的**Debug**选项里面选择**Serial Wire**，以防止之后无法下载程序。<br>

5. 设置好后，在顶侧**Project Manager**中，左侧选择**Code Generator**，为"**Generate peripheral initialization as a pair of .c/.h file per peripheral**"打勾，从而将外设初始化从`main.c`中分离出来。<br>

6. 左侧选择**Project**，在**Project Settings**中，**Project Name**输入项目名称，然后在**Toolchain/IDE**中选择：<br>
    - **STM32CubeIDE**以在STM32CubeIDE中编辑，<br>
    - **CMake**以在CLion/VSCode中编辑，<br>
    - **MDK-ARM**以在Keil/VSCode中编辑；<br>

7. 完成所有操作后，右上角**Generate Code**以生成项目代码。<br><br>

# 编辑代码
主程序位于`\Core\Src\main.c`：<br>

## 点灯大师
<big><strong>目的：使LED_Red规律闪烁。</strong></big><br>
在~~mian~~main函数的`while(1)`死循环中，在`/* USER CODE BEGIN 3 */`和` /* USER CODE END 3 */`之间，我们写入：<br>
```c
HAL_GPIO_WritePin(LED_Red_GPIO_Port, LED_Red_Pin, GPIO_PIN_SET);
```
<small>若不在此安全区内输入，后续若用CubeMX更改配置并重新生成代码，安全区外的代码会被覆写并丢失</small>
其中：<br>
- `HAL_GPIO_WritePin`函数：为指定的GPIO口写入高电平（`GPIO_PIN_SET`）/低电平（`GPIO_PIN_RESET`）；<br>
- `LED_Red_GPIO_Port`参数：LED_Red对应的GPIO口的组别；<br>
- `LED_Red_Pin`参数：LED_Red对应的GPIO口的编号；<br>
- `GPIO_PIN_SET`参数：设置为**高电平**，相反的，`GPIO_PIN_RESET`设置为**低电平**。<br>

在这段语句之后，写入：
```c
HAL_Delay(500);
```
其中：<br>
- `HAL_Delay`函数：指示等待；<br>
- `500`参数：等待500毫秒。<br>

在这两段语句后，继续写入：
```c
HAL_GPIO_WritePin(LED_Red_GPIO_Port, LED_Red_Pin, GPIO_PIN_RESET);
HAL_Delay(500);
```
从而让红色LED实现：“点亮500ms->熄灭500ms”的循环。<br>
之后，下载程序到单片机，按下单片机上的 **RST（Reset）** 按键，可以看到红色LED按照期望在闪烁。<br>
## 按键控制
<big><strong>目的：按下KEY1期间LED_Green点亮，按下KEY2切换LED_Red的亮灭。</strong></big><br>
使用以下代码用于获取按键按下情况：<br>
```c
HAL_GPIO_ReadPin(KEY1_GPIO_Port, KEY1_Pin);
```
其中：<br>
- `HAL_GPIO_ReadPin`函数：用于捕获指定GPIO口的高低电平，要注意返回值是`GPIO_PIN_SET`或`GPIO_PIN_RESET`而不是`1`或`0`；<br>
- `KEY1_GPIO_Port`参数：与`LED_Red_Pin`一样，`KEY1`对应的GPIO口的组别；<br>
- `KEY1_Pin`参数：与`LED_Red_Pin`一样，`KEY2`对应的GPIO口的编号。<br>

写入以下代码块：<br>
```c
if(HAL_GPIO_ReadPin(KEY1_GPIO_Port, KEY1_Pin)==GPIO_PIN_RESET){
    HAL_GPIO_WritePin(LED_Green_GPIO_Port, LED_Green_Pin, GPIO_PIN_SET);
}
else{
    HAL_GPIO_WritePin(LED_Green_GPIO_Port, LED_Green_Pin, GPIO_PIN_RESET);
}
```
这段代码的功能是：在KEY1按下期间，使绿色LED变亮，反之，若KEY没有被按下，则将其熄灭。<br><br>
写入以下代码块：<br>
```c
if(HAL_GPIO_ReadPin(KEY2_GPIO_Port, KEY2_Pin)==GPIO_PIN_RESET){
    HAL_Delay(10);
    if (HAL_GPIO_ReadPin(KEY2_GPIO_Port, KEY2_Pin)==GPIO_PIN_RESET)
    {
        HAL_GPIO_TogglePin(LED_Red_GPIO_Port, LED_Red_Pin);
        while (HAL_GPIO_ReadPin(KEY2_GPIO_Port, KEY2_Pin)==GPIO_PIN_RESET) {}
    }
}
```
这段代码的功能是：按下KEY2时，点亮红色LED，再次按下KEY2时，熄灭红色LED，实现一个“切换”的功能。<br>
- `HAL_GPIO_TogglePin`函数：<br>
    用于切换指定GPIO口的输出电平，若当前为高电平，则改为低电平；若当前为低电平，则改为高电平。<br>
- `HAL_Delay(10);`的作用：<br>
    按键在被按下的初期，电平会产生抖动，设置10ms的延时，若10ms后仍是低电平，则确定按键被按下，执行之后操作。<br>
- `while (HAL_GPIO_ReadPin(KEY2_GPIO_Port, KEY2_Pin)==GPIO_PIN_RESET) {}`的作用：<br>
    等待松开KEY2，防止重复触发。<br>

<small>实际上，这是非常屎山的写法，在之后学习更多内容后，可以对其进行改良。</small><br><br>

# GPIO的输入输出模式
<small>[参考视频](https://www.bilibili.com/video/BV1zG4y1K78S)</small><br>

## 推挽输出（Output Push Pull）
P-MOS和N-MOS协同工作：
<img src="/images/posts/STM32/STM32-1/pushpull.png" width="300" height="150">

### 当输出高电平时：
P-MOS被激活，N-MOS被关闭，此时VDD（3.3V）与IO引脚联通，输出高电平：
<img src="/images/posts/STM32/STM32-1/pushpull-set.png" width="300" height="150">

### 当输出低电平时：
N-MOS被激活，P-MOS被关闭，此时VSS（0V）与IO引脚联通，输出低电平：
<img src="/images/posts/STM32/STM32-1/pushpull-reset.png" width="300" height="150"><br>

## 开漏输出（Output Open Drain）
只有N-MOS进行工作，P-MOS一直处于断开状态：
<img src="/images/posts/STM32/STM32-1/opendrain.png" width="300" height="150">

### 当输出高电平时：
N-MOS被关闭，形成断路，无法形成回路：
<img src="/images/posts/STM32/STM32-1/opendrain-set.png" width="300" height="150">

### 当输出低电平时：
N-MOS被激活，与外侧的电压形成回路，产生电流：
<img src="/images/posts/STM32/STM32-1/opendrain-reset.png" width="300" height="150"><br>

## 推挽输出和开漏输出的区别：
||推挽输出|开漏输出|
|:-|:-|:-|
|优点|可以直接输出3.3V|配合外部电路，输出更加灵活|
|缺点|只能输出芯片的3.3V|高电平实际上是高阻态，无法输出电流|
<br>

## 复用推挽输出和复用开漏输出：
根据控制来源的不同（例如片上外设的复用功能输入），STM32将两种输出模式细分出复用推挽输出和复用开漏输出。<br>

## 上拉输入(Pull-up)
启用与VDD（3.3V）相接的电阻：
<img src="/images/posts/STM32/STM32-1/pullup.png" width="300" height="150">

## 下拉输入(Pull-down)
启用与VSS（0V）相接的电阻：
<img src="/images/posts/STM32/STM32-1/pulldown.png" width="300" height="150">

## 浮空输入(No pull-up and no pull-down)
两个电阻都不会启用，可依赖外部电路实现更多可能性：
<img src="/images/posts/STM32/STM32-1/nopullupandnopulldown.png" width="300" height="150"><br>

## 模拟输入
在施密特触发器之前产生分支，读取模拟信号，即输入电压的具体数值：
<img src="/images/posts/STM32/STM32-1/analoginput.png" width="300" height="150">

## 复用功能输入
在施密特触发器之后，读取的是数字信号（0或1）：
<img src="/images/posts/STM32/STM32-1/reusefunctioninput.png" width="300" height="150">
- 施密特触发器：在电压高于**高参考电压**时输出高电平，在电压低于**低参考电压**时输出低电平，在**高参考电压**和**低参考电压**之间则保持当前电平信号不变：

# 总结
## 相关函数：<br>
```c
HAL_GPIO_WritePin(GPIO_TypeDef *GPIOx, uint16_t GPIO_Pin, GPIO_PinState PinState);
```
1. `HAL_GPIO_WritePin`：用于向指定的GPIO口输出高电平/低电平<br>
- `GPIO_TypeDef *GPIOx`参数：指定GPIO口对应的组别；<br>
- `uint16_t GPIO_Pin`参数：指定GPIO口对应的编号；<br>
- `GPIO_PinState PinState`参数：`GPIO_PIN_SET`则设置为**高电平**，`GPIO_PIN_RESET`则设置为**低电平**。<br><br>

```c
HAL_Delay(uint32_t Delay);
```
2. `HAL_Delay`：用于等待指定的时长<br>
- `uint32_t Delay`：等待的时长（单位：毫秒）。<br><br>

```c
HAL_GPIO_TogglePin(GPIO_TypeDef *GPIOx, uint16_t GPIO_Pin)
```
3. `HAL_GPIO_TogglePin`：用于切换指定的GPIO口的输出电平<br>
- `GPIO_TypeDef *GPIOx`参数：指定GPIO口对应的组别；<br>
- `uint16_t GPIO_Pin`参数：指定GPIO口对应的编号；<br><br>

```c
HAL_GPIO_ReadPin(GPIO_TypeDef *GPIOx, uint16_t GPIO_Pin);
```
4. `HAL_GPIO_ReadPin`：用于读取指定的GPIO口的输出电平<br>
- `GPIO_TypeDef *GPIOx`参数：指定GPIO口对应的组别；<br>
- `uint16_t GPIO_Pin`参数：指定GPIO口对应的编号；<br><br>

## 输入输出模式：
- 推挽输出，开漏输出<br>
- 复用推挽输出，复用开漏输出<br>
- 上拉输入，下拉输入，浮空输入<br>
- 模拟输入，复用功能输入<br>
