<big><strong>NVIC(Nested Vectored Interrupt Controller) ，嵌套向量中断控制器，是ARM Cortex-M系列微控制器内核中集成的一个关键组件，用于管理中断和异常。</strong></big><br>

# 在芯片配置中启动外部中断
1. 左键芯片引脚，在功能中选择**GPIO_EXTIxx**（以PB12和GPIO_EXTI12为例）：<br>
<img src="/images/posts/STM32/STM32-2/enableinterrupt.png" width="400" height="400">

2. 在左侧选择**System Core**展开后选择**SYS**，在其中的**GPIO**选项可以调整GPIO口的具体信息：<br>
- `GPIO mode`：触发中断的模式，例如`External Interrupt Mode with Falling edge trigger detection`指的是**下降沿触发**，即由高电平变为低电平时触发中断；<br>

3. 配置好其他内容后，生成代码。<br>

# 编辑代码启用中断
<big><strong>目的：LED_Red规律闪烁，按下KEY1实现切换LED_Green亮灭。</strong></big><br>

先在main函数的`while(1)`中写入以下程序，实现红灯规律闪烁：<br>
```c
HAL_GPIO_WritePin(LED_Red_GPIO_Port, LED_Red_Pin, GPIO_PIN_SET);
HAL_Delay(500);
HAL_GPIO_WritePin(LED_Red_GPIO_Port, LED_Red_Pin, GPIO_PIN_RESET);
HAL_Delay(500);
```
中断函数位于`\Src\stm32f1xx.c`中的`void EXTI15_10_IRQHandler(void)`：<br>
```c
void EXTI15_10_IRQHandler(void)
{
  /* USER CODE BEGIN EXTI15_10_IRQn 0 */

  /* USER CODE END EXTI15_10_IRQn 0 */
  HAL_GPIO_EXTI_IRQHandler(KEY1_Pin);
  /* USER CODE BEGIN EXTI15_10_IRQn 1 */

  /* USER CODE END EXTI15_10_IRQn 1 */
}
```
由于我们设置的`GPIO mode`是`External Interrupt Mode with Falling edge trigger detection`，在KEY1被按下时，会从高电平变为低电平，从而触发中断函数`EXTI15_10_IRQHandler()`。<br>
在`/* USER CODE BEGIN EXTI15_10_IRQn 0 */`和`/* USER CODE END EXTI15_10_IRQn 0 */`之间写入以下代码：<br>
```c
HAL_GPIO_TogglePin(LED_Green_GPIO_Port, LED_Green_Pin);
```
下载程序后，KEY1基本可以实现绿灯的切换，但是多次尝试切换后，还是会发现有时不灵敏，这是因为硬件消抖还是不足以完全清除抖动，因此我们需要为其添加软件消抖：<br>
```c
HAL_Delay(10);
if(HAL_GPIO_ReadPin(KEY1_GPIO_Port, KEY1_Pin)==GPIO_PIN_RESET){
  HAL_GPIO_TogglePin(LED_Green_GPIO_Port, LED_Green_Pin);
}
```
下载程序后按下KEY1，我们却发现程序一动不动，甚至LED_Red都不再闪烁了。<br>
这是因为发生了多个中断的**优先级**问题，`HAL_Delay()`函数依赖于`Time base:System tick timer`的中断，但是此中断的优先级低于`EXTI line[15:10] interrupts`，因而不能在`EXTI15_10_IRQHandler`中执行。<br>
解决方法如下：<br>
1. 回到CubeMX中，在左侧的**System Core**中找到**NVIC**，修改各中断的优先级，我们让`Time base:System tick timer`的优先级高于`EXTI line[15:10] interrupts`<br>
<small>注：数字越小，代表其优先级越高</small><br>
<img src="/images/posts/STM32/STM32-2/nvic.png" width="400" height="400">
2. 完成修改后，重新生成代码，下载程序，便可以发现目的已经达成：LED_Red规律闪烁，按下KEY1切换LED_Green亮灭。<br>
> [!caution]
> 不建议在中断中使用`HAL_Delay()`的**阻塞式延时**，对于按键中断，在之后会有更加巧妙的方法来实现。

## 边沿检测电路
**上升/下降沿触发选择寄存器**：决定是检测到上升沿向后传递高电平信号，还是检测到下降沿向后传递高电平信号，或者二者都。<br>
配置`GPIO mode`，实际上就是在配置**上升沿触发选择寄存器**和**下降沿触发选择寄存器**。<br>
## 请求挂起寄存器
接受到**边沿检测电路**发送来的高电平时，其对应位会被置1。<br>
## 中断屏蔽寄存器
配置GPIO引脚为`GPIO_EXTIxx`时，实际上就是在配置**中断屏蔽寄存器**的对应位置1。<br>
## NVIC（嵌套向量中断控制器）
通过**与门**和**请求挂起寄存器**，**中断屏蔽寄存器**相连，当且仅当二者都置1时，才向后发送高电平信号。<br>
掌管中断向量表，即由名称指向中断处理函数。
|名称|处理向量|
|:-|:-|
|EXTI0|EXTI0_IRQHandler|
|EXTI1|EXTI1_IRQHandler|
|EXTI2|EXTI2_IRQHandler|
|EXTI3|EXTI3_IRQHandler|
|EXTI4|EXTI4_IRQHandler|
|EXTI9_5|EXTI9_5_IRQHandler|
|EXTI15_10|EXTI15_10_IRQHandler|

<br>
<big>例如，配置为`EXTI12`且**下降沿触发**的`KEY1`被按下，会依次发生以下过程：</big>

1. 生成代码阶段：
    - **上升沿触发选择寄存器**的第12位被置0；
    - **下降沿触发选择寄存器**的第12位被置1；
    - **中断屏蔽寄存器**的第12位被置1。
2. 经过输入驱动器处理的PB12的电平信号经由输入线进入到**边沿检测电路**；
3. **边沿检测电路**检测到高电平转化为低电平；
4. **边沿检测电路**向后发送高电平信号；
5. **请求挂起寄存器**接受到高电平信号，将第12位置1；
6. **中断屏蔽寄存器**和**请求挂起寄存器**的第12位均被置1，通过**与门**向后输出高电平信号；
7. **NVIC**找到中断向量`EXTI15_10`，执行函数`EXTI15_10_IRQHandler`；
8. 执行函数`HAL_GPIO_EXTI_IRQHandler()`，将**请求挂起寄存器**的第12位置0，终止中断执行，防止中断函数`EXTI15_10_IRQHandler`被反复执行。

## 中断优先级
分为**抢占优先级（Preemption Priority）** 和 **响应优先级（Sub Priority）**。:spoiler[为什么不是叫**次优先级**]<br>
在STM32中，每个中断向量都有4个二进制位来存储中断优先级信息，我们可以分别为**抢占优先级**和**响应优先级**分配不同的位数，默认地，会为**抢占优先级**分配4位，而为**响应优先级**分配0位，此时每个中断向量的**抢占优先级**可设置为0-15，而**响应优先级**只能是0。不过**响应优先级**的实际作用不大，一般保持默认设置为**抢占优先级**分配4位即可。
## 两个中断同时发生时
- 优先比较**抢占优先级**；
- 若**抢占优先级**相同，则再比较**响应优先级**；
- 若**响应优先级**也相同，则按照中断向量表中顺序决定执行顺序。
## 在其一中断执行中，接受到另一中断的信号
<big>仅比较**抢占优先级**：</big>
- 若后者中断B**抢占优先级**更高，则会中断当前执行的中断A，优先执行中断B，待中断B完成后，继续完成中断A。
<img src="/images/posts/STM32/STM32-2/irq1.png" width="400" height="300">
- 若后者中断B**抢占优先级**更低，则当前的中断A会继续执行，执行完中断A之后，再执行中断B。
<img src="/images/posts/STM32/STM32-2/irq2.png" width="400" height="300">

在刚才的项目中，设置`Time base:System tick timer`的优先级高于`EXTI line[15:10] interrupts`，便是为了系统计时的中断能够在外部中断执行过程中抢占执行，从而为`HAL_Delay()`函数提供时间基准。
